## Markor 闪退根因定位与修复（崩溃日志确认）

### 崩溃日志
用户提供了 4 份崩溃日志（crash_2026-07-02_18-33-07/12/20/23.log），设备为三星 SM-X800（Android 16 API 36），所有日志堆栈完全一致：

```
java.lang.ClassCastException: android.widget.FrameLayout cannot be cast to android.widget.LinearLayout
    at GsFileBrowserListAdapter$FilesystemViewerViewHolder.<init>(GsFileBrowserListAdapter.java:914)
```

### 根因
布局文件 `opoc_filesystem_item.xml` 的根元素是 `<FrameLayout>`（Apple 风格重设计时改的），但 Java 代码 `GsFileBrowserListAdapter.FilesystemViewerViewHolder` 中将 `itemRoot` 声明为 `LinearLayout`。`findViewById` 返回 FrameLayout 后赋值给 LinearLayout 变量，直接抛出 ClassCastException。

此崩溃在文件浏览器 RecyclerView 首次创建 ViewHolder 时触发，即授权后进入 MainActivity 的 Notebook Tab 时立即闪退。

### 修复
- 文件：`app/src/main/java/net/gsantner/opoc/frontend/filebrowser/GsFileBrowserListAdapter.java`
- 将 `final LinearLayout itemRoot` 改为 `final FrameLayout itemRoot`
- 添加 `import android.widget.FrameLayout`
- 验证所有 `itemRoot` 使用处（setPadding、setContentDescription、setTag、setOnClickListener、setOnLongClickListener）均为 View 基类方法，FrameLayout 完全兼容
