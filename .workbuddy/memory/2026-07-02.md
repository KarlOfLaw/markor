## Markor 闪退根因定位与修复（崩溃日志确认）

### 崩溃日志
用户提供了 4 份崩溃日志（crash_2026-07-02_18-33-07/12/20/23.log），设备为三星 SM-X800（Android 16 API 36），所有日志堆栈完全一致：

```
java.lang.ClassCastException: android.widget.FrameLayout cannot be cast to android.widget.LinearLayout
    at GsFileBrowserListAdapter$FilesystemViewerViewHolder.<init>(GsFileBrowserListAdapter.java:914)
```

### 根因
布局文件 `opoc_filesystem_item.xml` 的根元素是 `<FrameLayout>`（Apple 风格重设计时改的），但 Java 代码 `GsFileBrowserListAdapter.FilesystemViewerViewHolder` 中将 `itemRoot` 声明为 `LinearLayout`。`findViewById` 返回 FrameLayout 后赋值给 LinearLayout 变量，直接抛出 ClassCastException。

此崩溃在文件浏览器 RecyclerView 首次创建 ViewHolder 时触发，即授权后进入 MainActivity 的 Notebook Tab 时立即闪退。

### 修复
- 文件：`app/src/main/java/net/gsantner/opoc/frontend/filebrowser/GsFileBrowserListAdapter.java`
- 将 `final LinearLayout itemRoot` 改为 `final FrameLayout itemRoot`
- 添加 `import android.widget.FrameLayout`
- 验证所有 `itemRoot` 使用处（setPadding、setContentDescription、setTag、setOnClickListener、setOnLongClickListener）均为 View 基类方法，FrameLayout 完全兼容

## 自定义 Markdown 预览主题 Bug 修复 + CSS 导入功能

### 诊断：CSS 冲突导致主题代码背景色始终被覆盖

根因：`MarkdownTextConverter.CSS_INLINE_CODE` 硬编码 `background-color: #D2D2D880`，由于该常量在 `head` 中注入 HTML 的时机晚于 `themeCss`，导致所有非 Auto 主题的 code 背景色被硬编码颜色覆盖。

### 修复
1. **`MarkdownTextConverter.java`** — 从 `CSS_INLINE_CODE` 移除 `background-color`，只保留 padding/border-radius/overflow-wrap
2. **`TextConverterBase.java`** — 为 Auto 主题（`HTML002_HEAD_WITH_STYLE_LIGHT/DARK`）补充 `:not(pre) > code` 规则，确保默认主题也有正确的 code 背景色
3. **`TextConverterBase.java`** — 在 `putContentIntoTemplate()` 中注入自定义 CSS 文件内容

### 新增功能：自定义 CSS 文件导入
- 新增设置项 `Settings → View Mode → Custom Preview CSS File`（Inject 区域）
- 用户输入 CSS 文件绝对路径，渲染时自动读取并注入到 HTML head
- 如果文件内容不含 `<style>` 标签，自动包装
- 注入时机：built-in 主题 CSS 之后、用户 Inject → head 之前

### 重构：Typora 风格主题系统（用户要求）
将自定义 CSS 从"追加注入"模式改为**完整的主题切换**模式，类似 Typora：
- **`arrays.xml`** — 主题下拉列表新增 `Custom CSS` 选项（第 8 项），键值为 `"custom"`
- **`strings.xml`** — 添加 `markdown_preview_theme_custom` 显示名，更新设置项描述
- **`TextConverterBase.java`** — `getPreviewThemeCss()` 新增 `"custom"` 分支：读取用户 CSS 文件并返回作为完整主题；`putContentIntoTemplate()` 中当主题为 `"custom"` 时跳过重复的 `getCustomMarkdownCssContent()` 注入
- **使用流程**：设置 → View Mode → Markdown Preview Theme → 选 "Custom CSS"；然后在 Inject 区设置 CSS 文件路径。预览完全由用户 CSS 文件控制
